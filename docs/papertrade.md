# SayuStock AI 模拟盘 · 用户文档

> 每个群一个 AI 模拟账户（默认 100w 现金），早柚会在 A 股交易日每 30 分钟看一次盘，自动模拟买卖并用人格口吻推群。
>
> **设计哲学**：AI 操盘是**完全自主**的——用户只能初始化一次 + 查看状态，**不能干预 AI 的行为**。不能关停、调模式、添加自选股、强制决策、清空数据。AI 自己决定买什么卖什么。

## 快速开始

```
AI操盘初始化           # 开户（默认 100w 现金，平衡模式）— 整个流程唯一需要"动手"的命令
AI操盘初始化 2000000   # 自定义初始资金（1w~1亿）
AI操盘查看             # 看账户 + 持仓 + 当日盈亏图
AI操盘收益 月           # 看本月盈亏
AI操盘记录              # 看最近 20 笔交易
```

## 早柚会自动做什么

用户**不需要**发任何命令，AI 在每个交易日会自动：
- **每 30 分钟**（9:30-11:30 / 13:00-15:00）扫描自选池（≤50 只），结合技术面 + 基本面 + 舆情 + 风控评分（-1.0~+1.0），自动 buy / sell / hold
- **每个交易日 15:35** 写当日净值快照
- **每月 1 日 09:00** 出月度复盘报告
- 决策播报用人格口吻推群，附完整 reasoning

> 用户除了初始化和看数据，**不需要做任何事**。AI 操盘是观察型功能，不是控制型。

## 风险控制（AI 内部规则，**用户不能调整**）

| 规则 | 平衡（默认） | 激进 | 保守 |
|------|------|------|------|
| 单只最大仓位 | 25% | 40% | 15% |
| 单日最大交易次数 | 6 | 12 | 3 |
| 单笔止损 | -8% | -12% | -5% |
| 总回撤熔断 | -20% | -30% | -12% |
| 现金缓冲（buy 时） | 5% | 0% | 15% |
| 最大持仓数 | 8 | 12 | 5 |
| 同只 24h 加仓次数 | 1 | 2 | 1 |
| 强买 score 门槛 | 0.30 | 0.25 | 0.35 |
| 试探仓 score 门槛 | 0.10 | 0.08 | 0.15 |

> **高现金（80%+）是合法状态**——信号弱时 AI 主动持币，不强求满仓。

## 决策评分（-1.0 ~ +1.0）

| 类别 | 权重 | 关键规则 |
|------|------|----------|
| 技术面 | 40% | MACD 金叉 +0.20 / 死叉 -0.20；RSI 超卖/超买 ±0.10；均线多头/空头 ±0.10；CMF 强流入 +0.10；量比 > 2.0 +0.05；换手率健康 +0.05 |
| 基本面 | 30% | ROE > 15% +0.10；营收同比 > 20% +0.08；净利同比 > 30% +0.07；PE < 行业中位 × 0.7 +0.10；负债率 > 70% -0.10 |
| 舆情 | 15% | 正面新闻 ≥ 3 且 > 负面 × 2 → +0.15；业绩预增 +0.05；减持/利空 -0.05 |
| 波动率调整 | ±15% | ATR > 5% → score × 0.7；ATR < 1.5% → score × 1.1 |

## 候选池来源（5 路合并，50 只上限）

```
P0 持仓（≤ 20）        ← 当前持仓必分析
P1 内部池（≤ 20）      ← AI 上次决策的"持续关注"
P2 板块龙头（≤ 15）    ← 当日涨幅 TOP3 板块 × 5 只
P3 大盘热股（≤ 10）    ← 东方财富热股 TOP10
P3 新闻相关（≤ 10）    ← 雪球 7x24 提及的股票
```

> 单心跳成本：50 只 × 4 API = 200 次调用（asyncio.gather 并发 ≈ 5~10 秒）。

## 撮合规则（A 股真实费率）

- **价**：东财最新价（f43）立即成交
- **量**：100 股整手，向下取整
- **佣金**：max(成交额 × 0.00025, 5 元)
- **印花税**：卖出时 成交额 × 0.0005
- **现金不足**：自动降一档到能买得起的最大整手
- **持仓不足**：自动截断到实际可卖量

## 早柚主人格能做什么

| 群友说 | 早柚做什么 |
|--------|-----------|
| "模拟盘初始化" | **直接调 trigger `send_init_command`** —— 6 步全跑（DB + Kanban + APScheduler + bind + kick） |
| "模拟盘查看" | 调 papertrade_account_query + papertrade_position_list → 拼成图 |
| "你为啥买茅台？" | 调 list_my_kanban_tasks 找树 → artifact_get_recent 拿决策 reasoning → papertrade_trade_list 拿交易记录 → 用早柚口吻回答 |
| "现在还持有啥？" | 调 papertrade_position_list + papertrade_account_query → 拼成图 |
| "我的账户怎么样？" | 调 papertrade_account_query + papertrade_trade_list(limit=10) |

> 早柚"知道"这件事：插件通过 `ai_entity(KnowledgeBase("sayustock_papertrade_guide", ...))` 把操作指南注册为 RAG 知识库，群友问到 AI 操盘相关时早柚会主动召回。同时通过 `ai_alias("AI操盘", ["AI模拟盘", "虚拟盘", ...])` 把"AI操盘"等关键词路由到 papertrade 工具族。
>
> **工具分层**（无重叠）：主 persona 只能调 7 个 common 工具（4 读 + 3 helper），写操作 4 个只在子代理（visible_when）可见；"建账户"只走 trigger `send_init_command`，没有任何捷径。

| 命令 | 权限 |
|------|------|
| 初始化（AI操盘初始化） | 群主/管理员（user_pm <= 1） |
| 查询（查看、收益、记录） | 任何人 |

**实现**：基于 `ev.user_pm`：
- `user_pm == 0` → SUPERUSER / 主人
- `user_pm == 1` → 群主/管理员
- `user_pm >= 2` → 普通成员
- 私聊默认 `user_pm == 0`（给最大权限）

## 常见问题

### Q1: 模拟盘是真实交易吗？
**不是**。100% 虚拟，AI 只动 SQLModel 表，不会触碰任何真实账户。

### Q2: AI 决策质量差能赚钱吗？
不能保证。AI 用技术面 + 基本面 + 舆情 + 风控综合判断，长期可能跑赢指数也可能跑输。真实投资请自己判断，本插件不提供投资建议。

### Q3: 为什么我的 AI 长期 80% 现金？
信号弱时 AI 主动持币观望。这是有意设计——"高现金 + 等待良机" 优于 "硬买套牢"。

### Q4: 为什么我看到群里没人在卖 AI 操盘播报？
- 确认 `AI操盘初始化` 已开户（`AI操盘查看` 应有内容）；
- 确认当前是交易时段（9:30-11:30 / 13:00-15:00）；
- 确认是否交易日（周末/节假日不跑）；
- WebConsole → Kanban 看板 看任务树状态。

### Q5: 怎么暂停 AI？
**你不能**。这是只读型功能，AI 操盘一旦初始化就是完全自主的。如果想关掉，让群主去 WebConsole → Kanban 把对应任务 disarm。

### Q6: 怎么换风控模式（平衡/激进/保守）？
**你不能**。模式在初始化时固定为 balanced。如果想换，先去 WebConsole → SayuPaperAccount 改 `mode` 字段。

### Q7: 怎么跨群查其他群的 AI 操盘？
`AI操盘查询 <group_id>`（限群主/管理员）。普通用户发会收到"无权限"提示。

### Q8: 怎么重置本群的 AI 操盘？
**你不能**。这是有意设计——避免误操作导致数据丢失。如确有需要，联系 SUPERUSER 通过 WebConsole 直接操作 SQLModel。

## 故障排查

| 现象 | 排查 |
|------|------|
| 群里收不到 AI 操盘播报 | 1) 确认 `AI操盘查看` 有内容（已开户）；2) 确认是交易时段；3) WebConsole Kanban 看板看任务树 |
| 决策明显错误 | 等待 30 分钟让 AI 自己纠正（不允许手动干预）；或看 WebConsole 日志 |
| 想换风控模式 | WebConsole → SayuPaperAccount 改 `mode` 字段 |

## 设计原则总结

| 原则 | 含义 |
|------|------|
| **完全自主** | AI 自己决定买什么卖什么，不接受外部命令干预 |
| **只读交互** | 用户只能看，不能改（除初始化外） |
| **风险优先** | 信号弱时主动持币，不强求满仓 |
| **人格透明** | 决策推群 + 用户可查 reasoning，AI 不是黑盒 |
| **错误容忍** | 决策错误由 AI 自己纠正（止损/止盈/熔断） |
